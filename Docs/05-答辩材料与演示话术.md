# 05 答辩材料与演示话术

答辩流程（见要求文档）：PPT介绍（5分钟）→ 源程序演示+讲解（10分钟）→ 答疑（10分钟），全组30分钟内完成。以下内容按这三段分别准备。

## 第一段：PPT大纲（5分钟）

1. **封面**：项目名"慧笔——基于LangChain+LangGraph的英语写作智能批改与个性化学习伴学智能体"，小组编号/名称，成员名单。
2. **立项背景**：中国英语学习者写作批改的痛点（成本高/反馈慢/无持续追踪），引出选题动机（参考`00-项目总览.md`"立项背景与动机"）。
3. **总体架构图**：直接放`01-系统架构与Agent设计.md`里的架构图和LangGraph状态图，讲清楚"多智能体"具体指哪几个Agent、各自干什么。
4. **技术亮点（重点讲，评委关注加分项）**：
   - 两条自训练评分模型（微调BERT系 + 自建BiLSTM，均非套壳调API）+ QWK指标对标业界标准；
   - 通用大模型（DeepSeek主力/GLM兜底）负责生成式反馈，模型路径分工明确；
   - 创意模块：学习进步仪表盘+个性化练习推荐的持续学习闭环。
5. **完成效果展示**：1~2张关键截图（反馈详情页、进步仪表盘）+ 真实指标——路径A(微调DistilBERT)测试集QWK=0.693，路径B(自建BiLSTM)测试集QWK=0.622，覆盖全部8个essay_set（见`03-模型训练与微调方案.md`）。
6. **分工**：四人角色与主要贡献（对照`04-四天开发计划与分工.md`的角色表）。

## 第二段：源程序演示脚本（10分钟）

按以下顺序演示，每步说清楚"现在在展示哪个Agent/哪个模块":

1. 打开Streamlit"提交批改"页 → 粘贴一篇准备好的示例作文 → 提交。
2. 演示过程中展示各Agent节点执行进度（若做了status展示，重点讲一遍"现在校验通过→正在检索评分细则→正在调用自训练模型打分（A:0.95、B:0.05非等权融合）→正在生成定性反馈→正在生成个性化建议"），这一步是让评委直观看到"这是多智能体协作，不是一个大Prompt糊出来的"。
3. 打开"反馈详情"页 → 展示量化评分、语法纠错列表、定性反馈、个性化修改建议，挑1~2条读出来，证明反馈是具体、可操作的，不是空话。
4. 切到"历史进步仪表盘" → 用提前准备好的、有多次提交记录的演示账号展示评分趋势折线图和弱项雷达图 → 讲清楚这是创意加分模块，区别于市面上"一次性打分"工具。
5. ~~打开"写作知识库问答"页，现场问一个语法问题~~——**这个独立页面前端还没接通（输入框是禁用状态）**，不要现场演示这一步；RAG检索能力已经在"提交批改"流程里真实用上了（第1~3步反馈里引用的评分细则片段就是RAG检索结果），讲解时说明"检索能力已经在批改流程里体现"即可，不要临场打开这个页面露怯。
6. 简要过一遍代码结构（`src/agents/`的Graph定义、`src/training/`里微调与自建两套训练脚本），证明"整个逻辑小组自己清楚"（对应要求文档"注意"第2条）。

**容灾预案**：现场网络/API不可用时，切换到提前录制好的演示视频/截图集，讲解词同上一套，不要临场改讲法导致和PPT对不上。

## 第三段：预设问答（10分钟）——评委常见追问与应对

| 可能的问题 | 建议回答方向 |
|---|---|
| 为什么评分不直接让大模型打分，要单独训练评分模型？ | 不是空口说"方差大"，有真实对比数据：同一批80篇作文，DeepSeek零样本直接打分的macro QWK只有**0.384**，我们训练的模型采用验证集选出的非等权融合（A:0.95、B:0.05）后在同样80篇上是**0.759**，差距接近两倍（`models/zero_shot_llm_eval.json`、`models/weighted_ensemble_eval.json`，详见`09`号文档第四节）。大模型不知道具体评分基准的分数分布/校准，零样本预测系统性偏低；自训练模型可以用真实人工评分数据训练、用QWK这类业界标准指标量化评估。 |
| 怎么证明微调/自建模型是你们自己做的，不是调了个现成模型/改了改Prompt？ | 展示训练日志/loss曲线/QWK随训练轮数变化的曲线，以及路径A vs 路径B的横向对比（见`03-模型训练与微调方案.md`评估要点），可以现场打开训练脚本讲关键代码逻辑，尤其是路径B（自建BiLSTM）完全从零训练、不依赖任何预训练权重这一点要讲清楚。 |
| "225原则"具体是怎么用在你们的训练里的？ | 已确认：225原则指训练循环的"2层循环(epoch/batch)+2个遍历对象+5个核心步骤(梯度清零→正向传播→损失计算→反向传播→参数更新)"这一基础结构，两条评分模型的训练脚本都严格遵循这个结构，并在此基础上叠加了学习率调度/早停/分层数据划分等增强，可以现场打开`src/training/`的代码对照讲解（见`03-模型训练与微调方案.md`）。 |
| 为什么要用LangGraph多智能体，而不是一条Prompt链（Sequential Chain）搞定？ | 多智能体的好处是每个Agent职责单一、可以独立替换/调试（比如换一个评分模型不影响反馈生成逻辑）、且有条件路由（无效输入直接短路返回，不浪费后续调用），这是团队协作开发时也更容易并行分工的原因，`04`号文档的4人分工正是沿着这几个Agent节点划分的。 |
| 知识库(RAG)里的内容是从哪来的，会不会有版权/数据来源问题？ | 评分细则(rubric)/题目(prompt)提取自ASAP-AES数据集本身自带的字段（数据集允许学术/教育用途），语法卡片是团队自行编写；数据集通过HuggingFace上的非gated镜像获取（因为没有Kaggle账号无法走官方下载渠道，内容与Kaggle官方一致，见`00-项目总览.md`「参考资料」），如果评委追问可以如实说明数据获取渠道。 |
| 为什么这套系统在essay_set 8上表现明显更差（QWK只有0.15~0.47）？ | 如实说明，且有真实诊断数据支撑（不只是"数据稀疏"四个字带过）：essay_set 8训练样本量只有**578条**，明显少于其他集合的1252~1440条；在essay_set 8全量测试集上算过残差(预测-真实)与真实分数的相关系数，达到**-0.73**（强负相关），说明模型对真实高分系统性低估、真实低分系统性高估，是训练样本少+分值范围宽(15-60)场景下回归模型典型的"向均值收缩"症状，不是代码bug。两条模型路径在这个集合上都是最弱的，说明结论一致、可信，也是很好的"诚实展示局限性"的答辩素材（详见`09`号文档）。 |
| 历史进步仪表盘的雷达图（分项评分）是真正的AI多维度评分吗？ | 如实说明：不是单独训练的多头模型。三个维度的初始值都等于整体分（微调+自建融合的输出），`grammar_check_node`会在此基础上用可观察的文本信号做启发式调整——language看真实检测到的语法错误密度，content看词汇丰富度(type-token ratio)，organization看段落数/句长结构。这是"有信号支撑的启发式近似"，不是"训练出来的多头预测"，主动讲清楚这个区别比被问到才承认更专业。 |
| 如果没有网络/API服务临时不可用怎么办？ | 已准备离线演示预案（提前跑好的示例结果与录屏），见本文档"容灾预案"和`08-部署操作手册.md`。 |
| 这个项目和小组另一位/其他成员已有的项目（如医疗类项目）有什么区别，会不会重复？ | 该项目是纯文本域的多智能体编排+NLP模型微调+教育科技场景，与既有的医学影像CV项目在技术栈和应用领域上都不同，是两个独立的技术方向（详见`00-项目总览.md`"立项背景与动机"第3点）。 |
| 加分项具体体现在哪？ | 明确对照`00-项目总览.md`的"需求与加分项对照表"逐条讲：微调模型+自建模型两条路径（要求6加分项，一次覆盖两个选项）、Streamlit部署（要求10加分项）、学习进步仪表盘的创意模块（申请10~15分创意加分）。**主动讲出来，不要等评委发现**。 |
