# 02 数据集与数据处理方案

## 数据来源

### 开源数据集：Kaggle ASAP-AES

- 来源：Kaggle "Automated Student Assessment Prize"（自动作文评分）竞赛数据集，自动作文评分（AES）领域公认的标准基准。
- 规模：约1.3万篇英语作文，划分为8个`essay_set`（题目集），涵盖议论文、回应型作文、叙事型作文三类文体，每篇长度约150~650词。
- 字段：`essay_id`、`essay_set`、`essay`（作文正文）、`domain1_score`（主评分，各essay_set的分值范围不同）、部分essay_set（如7、8）还带有`trait`分项评分（内容/结构/语言/词汇等维度）。
- 用途：作为微调评分模型的训练/验证/测试数据，同时作为RAG知识库中"范文库"的原始素材来源。
- **实际获取渠道（已确认）**：Kaggle官方下载需要账号登录/接受比赛协议，团队没有Kaggle账号，改用HuggingFace上"llm-aes"组织发布的非gated镜像（`src/data_pipeline/download.py`，内容与Kaggle官方一致，覆盖全部8个essay_set共12976条），走`HF_ENDPOINT=https://hf-mirror.com`下载。如果之后拿到Kaggle账号想换回官方渠道，把`training_set_rel3.tsv`放到`data/raw/`下，`clean.py`会自动优先使用它，两种来源都支持，详见`RUNNING.md`第3节和`07-数据处理与前端操作手册.md`。
- **实际清洗结果**：12976条原始数据清洗后剩12879条（剔除97条空评分/过短/重复样本），EDA图表已生成在`data/processed/eda/`。

### 自定义数据：RAG知识库素材（已从真实数据集提取，不再是占位示例）

评分细则(rubric)和题目(prompt)已经用`src/rag/build_rubric_docs.py`从下载好的数据集里提取生成`data/kb/rubric_essay_set_1~8.md`（覆盖全部8个essay_set的真实内容，不是团队编造的）；语法卡片仍是团队自行整理编写（目前只有3条示例，可持续补充）：

1. 评分细则（rubric）文本——参照ASAP官方评分说明+通用英语写作评分维度整理。
2. 语法规则卡片——中国学生常见英语写作错误规则说明，团队编写。
3. 范文标注——从ASAP-AES训练集中挑选各分数段代表性作文，团队人工写"好在哪里/差在哪里"的简评。

这部分满足要求文档"数据集：开源数据集和自定义数据集或者爬虫数据"里"自定义数据集"的部分。

## 数据清洗

| 问题 | 处理方式 |
|---|---|
| 编码错误/乱码 | 统一转UTF-8，过滤无法解码或包含大量非法字符的样本 |
| 评分缺失 | `domain1_score`为空的样本直接剔除，不做插补（避免引入伪标签污染评分模型训练） |
| 异常长度 | 过滤词数极端（如<20词的空白/占位作文，或明显超出该essay_set正常范围的异常长文本） |
| 匿名占位符 | ASAP-AES作文中人名/地名等被替换为`@PERSON1`、`@ORGANIZATION1`等占位符；统一替换为通用占位词（如`[NAME]`/`[ORG]`）或还原为通用泛称，避免模型学到无意义的占位符模式 |
| 重复样本 | 按`essay_id`去重，检查是否有内容高度相似的重复提交 |

## 数据分析与可视化（EDA）

产出一份EDA notebook（或脚本+图片集），至少包含：

1. **各essay_set的作文数量分布**（柱状图）——说明各题目集样本量是否均衡。
2. **各essay_set的评分分布**（分essay_set的直方图/箱线图）——不同题目集分值范围不同，需要分开看。
3. **作文长度（词数）与评分的散点图/相关性分析**——检验"是否写得越长分越高"这类朴素假设，为后续是否引入长度特征做依据。
4. **训练/验证/测试集的分布一致性检查**——确保划分后各子集的评分分布、essay_set比例大致一致，避免验证结果失真。
5. **（如使用trait分项数据）各维度分项评分之间的相关性热力图**——理解"内容/结构/语言"等维度是否高度相关，指导是否需要分别建模。

这一节直接对应要求文档"数据分析：针对训练集和验证集的数据进行分析以及可视化"。

## 数据预处理

1. **分数归一化**：由于各`essay_set`的分值范围不同（如0-12、0-30、0-60不等），需要将原始分数按each set的最小/最大值归一化到统一的[0,1]区间用于模型训练，推理后再反归一化回该题目集的实际分值区间展示给用户。
2. **文本分词**：使用所选预训练模型（如`bert-base-uncased`）配套的tokenizer，按模型最大输入长度做截断/填充（长作文超出512 token时的截断策略需要在文档里写清楚采用哪种——建议优先截断中间保留首尾，或简单截断尾部，二选一并说明理由）。
3. **数据集划分**：训练/验证/测试集划分，需按`essay_set`分层抽样，保证每个子集里各题目集比例一致，避免模型只在某几个题目集上学得好。具体划分比例见`03-模型训练与微调方案.md`中"225原则"一节的说明。
4. **类别不平衡处理**（如需要）：若某些essay_set的高分/低分样本比例严重失衡，评估是否需要过采样/加权损失，权衡后在训练方案里写明最终选择。
5. **面向真实应用场景的适配**：真实用户提交的是"任意主题的英语作文"，而不一定完全对应ASAP-AES的8个固定题目。预处理阶段需要设计一个"通用性回退"策略——当用户作文的主题与训练用的8个essay_set均不匹配时，评分模型仍需给出一个合理的通用评分（例如退化为跨题目集统一训练的通用维度模型，或者在Prompt里明确告知系统"此评分为通用参考值，非该题目专属校准分"）。这一步是保证模型"更适配真实应用场景、提高泛化能力"的关键设计点，对应要求文档"数据预处理"条目里的表述。
