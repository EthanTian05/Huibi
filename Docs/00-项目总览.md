# 00 项目总览

## 项目名称

- **中文名**：慧笔（HuiBi）
- **完整题目**：慧笔——基于LangChain+LangGraph的英语写作智能批改与个性化学习伴学智能体
- **英文**：HuiBi — A LangChain + LangGraph Multi-Agent System for English Writing Assessment and Personalized Coaching
- **建议的提交文件夹命名**（按要求格式`小组编号_小组名称_项目名称`）：
  `07_速通小分队_慧笔——基于LangChain+LangGraph的英语写作智能批改与个性化学习伴学智能体`
  根目录已改名为"7_速通小分队_慧笔"（简化版，未用完整长名），团队已手动执行完成。

## 一句话介绍

面向中国英语学习者（四六级/考研英语/雅思写作场景）的多智能体写作批改助手：提交一篇英语作文，系统给出**量化评分（自训练模型）+ 结构化定性反馈（多智能体协同生成）+ 个性化弱项练习推荐**，并持续追踪多次提交的进步曲线。

## 立项背景与动机

1. **真实需求**：四六级/考研英语/雅思写作是中国英语学习者最普遍的痛点之一——批改成本高（人工精批不可规模化）、反馈滞后（考试后才知道扣分点）、缺乏持续追踪（改完一篇不知道自己到底有没有进步）。这是一个有真实用户画像和商业逻辑的选题，符合"创新创业实训"的定位。
2. **选题差异化**：本次实训要求"整个班级不能出现重复的项目"，多数同类实训的示例方向集中在"本地知识库问答机器人""通用聊天助手"一类（见任务要求文档给出的范例）。选择"写作评估+学习陪伴"这一垂直教育场景，既满足LangChain+LLM+LangGraph的技术要求，又降低与同班其他组撞题的概率（**仍需团队实际核实同班选题，见TODO**）。
3. **和团队成员已有履历互补而非重复**：团队成员田溯开已有RetinaScope（医疗影像多标签筛查 + Streamlit + RAG问诊）项目背书，慧笔选择的是"纯文本域的多智能体编排 + 有监督微调评分模型 + 教育科技产品化"，与既有的"医学影像+CV"项目形成技术栈互补（全栈/RAG能力延续，但这次核心难点是LangGraph多智能体状态编排和NLP微调，而非CV），便于简历/答辩中讲"两个不同领域都能独立交付可用系统"的故事，而不是同一件事做两遍。
4. **数据集容易获取、质量有保障**：核心数据集Kaggle ASAP-AES（Automated Student Assessment Prize）是自动作文评分（AES）领域公认的标准基准数据集，公开、干净、有大量论文和baseline代码可参考（见下方参考资料），显著降低"数据集不好找/质量差"的项目风险。
5. **4天可落地**：核心链路难度可控——微调一个中等规模的评分模型、编排5个左右的Agent节点，都是有成熟公开教程可依循的任务，且四个子模块（数据/模型、Agent编排、前端部署、集成与文档）边界清晰、可并行推进，适合4人4天的强度。

## 目标用户与场景

- **主要用户**：准备四六级/考研英语/雅思写作的中国大学生。
- **核心场景**：学生提交一篇作文草稿 → 系统给出评分和结构化反馈 → 学生按建议修改 → 多次提交后系统展示进步趋势和下一步该练什么。
- **非目标（明确不做，避免范围蔓延）**：不做真实雅思/托福官方评分认证、不做人工客服兜底批改、不做多语种（仅英语写作）、不做移动端App。

## 技术栈总览

| 层次 | 选型 | 说明 |
|---|---|---|
| Agent编排 | LangGraph（StateGraph） | 状态在Agent节点间显式流转，支持条件路由 |
| LLM调用/工具/RAG链 | LangChain | ChatModel封装、Tool封装、Retriever封装 |
| 通用大模型 | DeepSeek V4 Flash（主力）+ GLM-4.7-Flash（免费兜底），OpenAI兼容接口调用 | 负责定性反馈生成、RAG问答、辅导建议；Key见本地`.env`，不进版本库 |
| 自训练评分模型·路径A | HuggingFace Transformers（BERT系，如`bert-base-uncased`/`distilbert-base-uncased`） | 小组微调，产出量化评分+多头分项评分，满足"预训练模型-微调" |
| 自训练评分模型·路径B | 自定义BiLSTM+Attention（不依赖预训练权重，从零训练） | 小组从零构建，满足"自定义构建模型"加分项，与路径A做横向对比 |
| 向量库 | Chroma（本地，轻量、无需额外服务） | 存储rubric/语法卡片/范文的embedding |
| Embedding模型 | 开源embedding模型（如`sentence-transformers`/BGE系，Day1按下载速度与效果确定具体型号） | 已确认走开源本地模型，不依赖大模型API的embedding接口，见`01-系统架构与Agent设计.md` |
| 数据存储 | SQLite | 用户历史提交与进步追踪，轻量、免安装 |
| 前端/部署 | Streamlit，SSH部署至自有服务器（`121.41.238.92`） | 与团队已有技能栈契合，开发速度快 |
| 数据分析 | pandas / matplotlib / seaborn | EDA与可视化 |

## 参考资料

以下开源项目/资料仅作**架构与基线思路参考**，团队理解其设计后自行实现，不直接搬运代码提交——尤其评分模型的微调训练部分，题目明确要求"必须自己完成"，答辩时需要能讲清楚每一处关键逻辑（对应要求文档"注意"第2、3条）。

- Kaggle ASAP (Automated Student Assessment Prize) 数据集与竞赛页面——自动作文评分领域的标准基准数据集来源。
- [Gaurav-Pande/AES_DL](https://github.com/Gaurav-Pande/AES_DL) — 基于BERT的ASAP作文评分实现，可用于理解"预训练模型+回归头"这一微调思路的baseline写法。
- [zkurtz/ASAP-AES-workbench](https://github.com/zkurtz/ASAP-AES-workbench) — 传统特征+word2vec的AES实现，可用于对比理解为什么选择Transformer微调而非纯特征工程。
- [langchain-ai/rag-research-agent-template](https://github.com/langchain-ai/rag-research-agent-template) — LangChain官方的RAG Agent模板，可参考其检索链与配置化写法。
- [junfanz1/Cognito-LangGraph-RAG](https://github.com/junfanz1/Cognito-LangGraph-RAG) — LangGraph实现的多步RAG（含相关性打分、网络兜底检索），可参考"条件路由+多节点状态图"的设计模式。
- [LangGraph官方Agentic RAG教程](https://docs.langchain.com/oss/python/langgraph/agentic-rag) — 官方文档，State Graph与检索Agent的标准写法。
